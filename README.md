## **1번째 미션 : RESTful API, DTO 학습하기**

### **개요**

RESTful API를 학습하면 클라이언트와 서버 사이의 명확한 계약과 일관된 리소스 설계로 확장성·유지보수성을 높일 수 있습니다.

DTO를 학습하면 계층 간 데이터 전달을 표준화해 불필요한 의존성을 줄이고 입력·출력 검증을 쉽게 할 수 있습니다.

## **2번째 미션 : API 명세서 작성하기 및 Swagger 사용해 문서화하기**

### **개요**

API 명세서 작성과 Swagger 문서화는 프론트엔드·백엔드·테스트팀이 따를 명확한 ‘계약’을 만들어 개발 충돌을 줄이며,

이를 통해 팀 간 원활한 소통과 빠른 통합 검증 및 연동을 가능하게 합니다.

## **소비자**

### **재고 검색**

해당하는 이름에 대한 물건의 정보를 보여준다.

물건의 정보에는 물건 이름, 물건 가격, 재고 수가 있다.

이름 당 물건은 1개뿐이라고 가정한다.

### **재고 구매**

장바구니에 담긴 물건을 구매한다.

아래의 정보를 반환해야 한다.

- 총 구매액
- 각 물건의 정보
    - 이름
    - 구매한 개수
    - 해당 물건에서 소비한 금액

## **관리자**

### **재고 등록**

새로운 물건을 쇼핑몰에 등록한다.

기존에 존재하는 물품을 시도할 경우, 에러를 반환한다.

물건을 등록할 때 위에서 말한 물건 정보 세 가지를 포함하면 된다.

### **재고 추가**

기존의 물품에 대해, 추가할 재고 수를 입력하여 재고를 늘린다.

추가하여 늘어난 최종 재고 수를 반환하면 된다.

### **물품 삭제**

입력으로 주어진 물품에 대해, 쇼핑몰에서 삭제한다.

입력은 1개 이상의 물품일 수 있다.

출력으로 현재 쇼핑몰에 남아있는 물품과 개수를 반환한다.

## **3번째 미션 : 쇼핑몰 페이지 Controller 계층 구현하기**

### **개요**

Controller 계층은 클라이언트의 HTTP 요청을 수신해 경로·쿼리·바디를 DTO로 매핑하고, 입력 유효성 검사 후 서비스 계층에 요청을 전달하는 역할을 합니다. 또한 서비스로부터 받은 결과를 적절한 HTTP 상태 코드와 응답 포맷으로 변환해 클라이언트에 반환하며, 비즈니스 로직은 직접 담지 않고 요청·응답 흐름 제어에 집중하는 역할을 합니다.
