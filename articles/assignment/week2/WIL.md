# 객체지향 분석
기준이 애매모호 하다  

## 어떤 객체로 나누면 좋을지

객체는 단 하나의 책임만 가져야 한다
기능담당

1. 아이템 객체
2. 저장소 객체 (데이터를 저장/검색할 장소를 만듦)
3. 서비스 객체 (여러 객체를 묶어서 처리하는 로직?)
4. 컨트롤러

## 어떤 책임을 가지면 좋을지

1. 아이템 객체 : 자신의 속성 보관
2. 저장소 객체 : 상품 저장, 삭제, 검색
3. 서비스 객체 : 예외 상황 처리, 중복 여부 판단, 주문시 재고 차감 , 총 금액 계산
4. 컨트롤러 객체 : HTTP 요청 파싱

## 어떻게 협력하여 문제를 해결할지
+ 소비자 재고 검색
    1. 컨트롤러: 입력받기 서비스 전달 (입력받기 책임)
    2. 서비스 : 저장소에서 조회 요청 결과 반환 (조회 요청)
    3. 저장소 : DB에서 조회 (조회)
    4. 아이템 : 데이터 제공 

+ 소비자 구매
    1. 컨트롤러: 입력받기 서비스 전달 (입력받기)
    2. 서비스 : 재고 확인 요청, 재고 차감 요청 , 금액 계산 (서비스로직,매서드 분리)
    3. 저장소 : 재고 조회, 반영 후 저장


+ 관리자 물품 등록 / 수정 / 삭제
    1. 컨트롤러: 입력받기 서비스 전달 
    2. 서비스 : 중복 확인을 위한 메서드, 저장 요청
    3. 저장소 : 재고 조회, 반영 후 저장

+ 관리자 물품 수정 
    1. 컨트롤러: 입력받기 서비스 전달 
    2. 서비스 : 존재 확인을 위한 메서드, 수정 후 저장 요청
    3. 저장소 : 재고 조회, 반영 후 저장

+ 관리자 물품 삭제
    1. 컨트롤러: 입력받기 서비스 전달 
    2. 서비스 : 존재 확인을 위한 메서드, 삭제 요청
    3. 저장소 : 재고 조회, 반영 후 저장
