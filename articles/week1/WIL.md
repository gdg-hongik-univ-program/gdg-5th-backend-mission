## 1주차 WIL

1주차에서는 API, dto 공부와 swagger를 이용한 api 문서화 미션을 했다. dto와 controller의 관계를 이론으로만 공부한것이 아니라 직접 코드를 짜보고 문서화를 통해 눈으로 확인하면서 어떤 과정으로 dto와 controller가 작동하는지 알 수 있었다.

코드를 짜면서 @{method}Mapping, @Getter, @AllArgsConstructor 등 어노테이션에 대해 처음 접했고 @{method}Mapping: HTTP {method} 요청이 이 주소로 오면 이 메서드를 실행하는 것, @Getter: 클래스에 있는 모든 필드(변수)의 get...() 메서드를 자동으로 만들어주는 것,  @AllArgsConstructor: 모든 필드를 받는 생성자 자동 생성이라는 어노테션을 새롭게 공부했다.

코드를 작성하면서 Spring에 대해 너무 모르고 있다는 느낌을 받았고 이를 해결하기 위해 빠른 시일 내에 김영한의 스프링 입문 강의를 들어야겠다고 생각했다.

모든 팀원이 Swagger 문서화 미션에서 막혀 팀 스터디 시간에 다같이 블로그도 찾아보고 서로 소통해서 문제를 해결 할 수 있었다. 혼자 하는것 보다 모르는 것이라도 팀 단위로 다같이 알아가니까 속도가 훨씬 빨랐고 서로 소통하면서 진행하니 재미도 있었다.

Spring도 처음, 깃허브도 처음이여서 모르는게 너무 많은 1주차였다. 아직 배워야ㄴ할게 너무 많은 상태라 더 많은 시간을 투자하고 공부해서 구글링이나 AI를 사용하지 않고도 프로젝트를 따라가는 실력을 키워야겠다고 다짐했다.
