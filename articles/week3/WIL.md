@Repository

데이터 접근 계층을 정의하는 어노테이션

역할 : 이 클래스가 데이터 접근 계층의 컴포넌트임을 Spring에게 알려준다. 주로 데이터베이스와의 상효작용(CURD)을 담당하는 클래스에 사용한다.

예외 변환 : @Repository의 가장 중요한 기능 중 하나는 데이터 접근 예외 변환이다. 어노테이션이 붙은 클래스에서 발생하는 JPA/Hibernate 관련 예외를 Spring 표준 예외 계층으로 변환해준다. 이렇게 하면 개발자가 특정 데이터베이스 기술에 종속되지 않고 일관되 ㄴ방식으로 예외를 처리할 수 있다.


@Entity

역할 : 이 클래스가 JPA에 의해 관리되는 엔티티 클래스임을 지정한다. 자바 객체가 데이터베이스의 테이블과 1:1로 매핑된다는 것을 의미한다.

필수 조건 : @Entity가 붙은 클래스는 반드시 테이블의 PK를 지정하는 @Id 어노테이션이 붙은 필드를 가지고 있어야 한다.

기능 : @Entity를 통해 JPA는 객체의 상태 변경을 감지하고, 자동으로 적절한 SQL 쿼리를 생성하여 데이터베이스에 반영한다.\\


@Transactional

역할 : 메서드나 클래스에 데이터베이스 트랜잭션 기능을 적용하도록 지시한다. 트랜잭션은 데이터 베이스 작업을 하나의 단위로 묶어, 전부 성공하거나 전부 실패하도록 보장하는 기능이다.
